Qu’ran Surah

We are going to use an API today that gives information about the Qu’ran. The text is broken up into 114 surah (similar to Chapters). We can get metadata about each of the surah by using an API defined as follows:

  • protocol: http
  • authority: api.alquran.cloud
  • path: v1/surah

There are no query parameters needed for this API call. In the block below, call this API using a cache directory and save the results of the parsed JSON data as an object named obj:

# Question 01
url_base <- modify_url("http://api.alquran.cloud/v1/surah")
res <- dsst_cache_get(url_base, cache_dir = "cache")
obj <- content(res, type = "application/json")

Now, in the next block, create a tabular dataset containing as much information as possible from the object. Use the techniques we saw today to figure out the returned objects structure:

# Question 02

Qu’ran Surah Text

The same service that we used above also provides the text of individual surah. This can be done with the HTTP API using the following parameter, where # is replaced by the surah number.

  • protocol: http
  • authority: api.alquran.cloud
  • path: v1/surah/#/en.asad

In the code below, grab the data for of the 2nd surah (“The Cow”) using this API and save the result of the parsed JSON data as obj:

# Question 04
url_base <- modify_url("http://api.alquran.cloud/v1/surah/2/en.asad")
res <- dsst_cache_get(url_base, cache_dir = "cache")
obj <- content(res, type = "application/json")

Again, create a tabular dataset out of obj, storing as much information as possible.
